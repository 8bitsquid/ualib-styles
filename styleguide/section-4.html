Icons 4.0

Uses the Font Awesome icon service. Only use Bootstrap glyphicons if Font Awesome does not provide the needed icon.

To load icons, add this head include <link href="//maxcdn.bootstrapcdn.com/font-awesome/4.2.0/css/font-awesome.min.css" rel="stylesheet">.

  • All icon classes must be paired with the fa class. (e.g., the fa-life-ring icon would require class="fa fa-life-ring")
  • Always use span instead of i element. Even though the Font Awesome docs use i, we want to avoid repurposing existing HTML elements for WCAG 2.0AA compliance.
  • Use sparingly in sub menus.
  • Scroll through the definitions below to determine which icon to use.
  • Read through Font Awesome's example docs to see what tricks this icon library offers.

Placement & Use 4.1

Navigation 4.1.1

When used in navigation (e.g., navs, navbars):

  • Always place icon to the left of the text
    • If icons are more readable on the right side, then all icons must appear on that side (no mixing of icon placement)
  • Always leave a space between the text and the icon element
  • Always use fixed-width icons by adding the fa-fw class
  • If part of the same navbar, either all or none of the nav elements should have icons.
