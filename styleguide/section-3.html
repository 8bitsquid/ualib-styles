Buttons 3.0

Usage rules and recommendations:

  • All buttons must use have the .btn class declared, regardless of what type of button is it.
  • Always use the primary button, unless otherwise stated in the styleguide.
  • Use <button type="button"> over <a href="#"> and <input type="button">, whenever possible.
  • Always declare a type attribute with your button.

Button Types 3.1

More often than not, you will only use the primary and default buttons. However, there are more options available for success, info, warning, and danger prompts or modals.

Primary Button 3.1.1

Always use this button, except in the following contexts:

  • Button appears on top of a low-value background color
  • Button appears on top of the primary brand color
  • Button appears inside a success, info, warning, or danger prompt or modal
                                                <button type="button" class="btn btn-primary">Primary</button>
<button type="button" class="btn btn-primary active">Primary Active</button>
<button type="button" class="btn btn-primary disabled">Primary Disabled</button>

Default Button 3.1.2

Usage:

  • Use when the primary button is not appropriate (e.g., on a top a red background or any low-value color with a saturation > 10).
  • Avoid using in prompts or modals
                                                <button type="button" class="btn btn-default">Default</button>
<button type="button" class="btn btn-default active">Default Active</button>
<button type="button" class="btn btn-default disabled">Default Disabled</button>

Success Button 3.1.3

Usage:

  • Inside a success prompt or modal
                                                <button type="button" class="btn btn-success">Success</button>
<button type="button" class="btn btn-success active">Success Active</button>
<button type="button" class="btn btn-success disabled">Success Disabled</button>

Button modifiers 3.2

There are several classes that will modify a buttons appearance. Two of these modifiers (.disabled and .active) are featured with the button types above.

Sizes 3.2.1

Utilize the .btn-lg, .btn-sm, or .btn-xs classes for different size buttons

                                                <p>
<button type="button" class="btn btn-primary btn-lg">Large button</button>
<button type="button" class="btn btn-default btn-lg">Large button</button>
</p>
<p>
<button type="button" class="btn btn-primary">Default button</button>
<button type="button" class="btn btn-default">Default button</button>
</p>
<p>
<button type="button" class="btn btn-primary btn-sm">Small button</button>
<button type="button" class="btn btn-default btn-sm">Small button</button>
</p>
<p>
<button type="button" class="btn btn-primary btn-xs">Extra small button</button>
<button type="button" class="btn btn-default btn-xs">Extra small button</button>
</p>

Block Buttons 3.2.2

Use the .btn-block to stretch the button to the width of its parent element.

                                                <div style="width: 300px; background-color: #ccc; margin: 0 auto; padding: 5px;">
   <button type="button" class="btn btn-primary btn-lg btn-block">Block level button</button>
   <button type="button" class="btn btn-default btn-lg btn-block">Block level button</button>
</div>
